安装

nvidia-container-toolkit安装指南
nvidia-container-toolkit安装指南2

  • 主要安装docker，docker-compose，nvidia-container-toolkit。

  • archlinux直接使用pacman安装即可

  • 启动前先修改docker镜像源sudo vim /etc/docker/daemon.json

  • 配置nvidia-container-toolkit：sudo nvidia-ctk runtime configure --runtime=docker

  • 启动docker服务sudo systemctl start docker，sudo systemctl enable docker

使用

快速入门简介

常用指令

  • docker ps查看正在运行的容器，docker ps -a查看所有容器
  • docker images查看镜像。
  • docker pull [镜像]从镜像源拉取镜像。
  • docker run [参数] [镜像]运行容器。常用参数有-d后台运行，-p端口映射，-v挂载卷，-i交互模式，-t终端模式，--gpu all启用gpu，--rm容器关闭后即删除容器。

例如：

  • docker stop [容器]停止容器。
  • docker rm [容器]删除容器。
  • docker rmi [镜像]删除镜像。

Dockerfile

Dockerfile是Docker镜像的构建文件，用于描述镜像的构建过程。Dockerfile中的指令包括：

  • FROM 指定基础镜像，例如FROM ubuntu:latest。
  • RUN 执行命令，例如RUN apt-get update && apt-get install -y python3。
  • COPY 将文件或目录复制到镜像中，例如COPY . /app。
  • WORKDIR 指定工作目录，例如WORKDIR /app
  • EXPOSE 指定容器的端口，例如EXPOSE 80。

EXPOSE 指定的端口只是声明一下，当使用docker run命令时，配合-P或--publish-all选项，Docker会自动将EXPOSE声明的端口映射到宿主机上的随机端口。例如，如果Dockerfile中包含EXPOSE 80，那么运行docker run -P时，Docker会为宿主机选择一个未被占用的端口（如49153），并将该端口与容器内的80端口建立映射。或者直接使用-p 选项指定端口映射，例如docker run -p 8080:80。

  • CMD 指定容器启动时执行的命令，例如CMD ["python", "app.py"]。

Dockerfile一般会放在项目的根目录下，然后使用docker build -t [镜像名]:[版本号] . 命令构建镜像。

docker-compose

docker-compose是一个用于定义和运行多容器Docker应用程序的工具。它允许我们使用YAML文件来描述应用程序的各个组件，包括它们的依赖关系和配置信息。

一般在多个容器之间进行通信时，需要使用docker-compose。详见快速入门简介。

安装docker插件、remote development插件包。
